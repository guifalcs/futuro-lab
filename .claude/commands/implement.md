## Processo:

1. **Analise** rapidamente o que precisa ser feito e quais arquivos serão afetados
2. **Verifique** componentes existentes em `frontend/src/app/shared/components/` antes de criar novos
3. **Implemente** as mudanças seguindo os padrões do projeto
4. Se criou componente Angular, **crie a story** no Storybook com todos os estados relevantes
5. Rode **lint** (`cd frontend && ng lint`) e corrija problemas
6. Rode os **testes relacionados** e corrija falhas
7. **PARE** e peça ao usuário para testar/validar
8. Após aprovação, faça **commit**: `tipo(escopo): descrição` em português
9. Adicione entrada no **CHANGELOG.md**: data, tipo, resumo, hash do commit

IMPORTANTE: Sempre pare para validação do usuário antes de commitar.
